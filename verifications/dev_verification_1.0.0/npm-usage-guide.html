Quick Start

Installation

bash 
npm install eigenvue
i Requires Node.js 18+. Zero runtime dependencies — everything is bundled.

Import

ESM (recommended) 
import { list, steps, show } from "eigenvue";
CommonJS 
const { list, steps, show } = require("eigenvue");

Your First Visualization (3 lines)

TypeScript 
import { show } from "eigenvue";

// Opens an interactive visualization of Binary Search in your browser
show("binary-search");

List All Available Algorithms

TypeScript 
import { list } from "eigenvue";

const algorithms = list();
console.log(`${algorithms.length} algorithms available`); // "17 algorithms available"

for (const algo of algorithms) {
  console.log(`  ${algo.id} — ${algo.name} (${algo.timeComplexity})`);
}

Generate Step-by-Step Data

TypeScript 
import { steps } from "eigenvue";

const result = steps("binary-search", {
  array: [1, 3, 5, 7, 9, 11, 13],
  target: 7
});

console.log(`Generated ${result.steps.length} steps`);

for (const step of result.steps) {
  console.log(`  Step ${step.index}: ${step.title} — ${step.explanation}`);
}
02

API Reference

Eigenvue exports three core functions and a version constant. All functions are synchronous.

list(options?)

returns AlgorithmInfo[]

Lists all available algorithms with their metadata. Optionally filters by category. Results are sorted by category, then by name.

ParameterTypeDescription
optionsListOptionsOptional. Pass { category } to filter results.
options.categoryAlgorithmCategoryOne of "classical" | "deep-learning" | "generative-ai" | "quantum"

Throws: Error if an invalid category string is provided.

Example 
import { list } from "eigenvue";

// All algorithms (17 total)
const all = list();

// Filter by category
const classical = list({ category: "classical" });       // 7 algorithms
const deepLearn = list({ category: "deep-learning" });   // 5 algorithms
const genAi     = list({ category: "generative-ai" });   // 5 algorithms

// Each AlgorithmInfo contains:
const algo = all[0];
console.log(algo.id);              // "backpropagation"
console.log(algo.name);            // "Backpropagation"
console.log(algo.category);        // "deep-learning"
console.log(algo.description);     // "Neural network training..."
console.log(algo.difficulty);      // "intermediate"
console.log(algo.timeComplexity);  // "O(n * m * k)"
console.log(algo.spaceComplexity); // "O(n * m)"

steps(algorithmId, inputs?)

returns StepSequence

Generates a complete step-by-step execution trace for an algorithm. Each step includes algorithm state, visual rendering actions, and source code highlights.

ParameterTypeDescription
algorithmIdstringAlgorithm identifier (e.g., "binary-search", "self-attention").
inputsRecord<string, unknown>Optional custom input parameters. Uses defaults from metadata if omitted.

Throws: Error if the algorithm ID is unrecognized or generation fails.

Example — Default Inputs 
import { steps } from "eigenvue";

const result = steps("binary-search");

console.log(result.formatVersion);  // 1
console.log(result.algorithmId);    // "binary-search"
console.log(result.inputs);         // { array: [...], target: ... }
console.log(result.steps.length);   // Number of execution steps
console.log(result.generatedAt);    // "2026-02-15T12:00:00.000Z"
console.log(result.generatedBy);    // "typescript"
Example — Custom Inputs 
const result = steps("binary-search", {
  array: [2, 4, 6, 8, 10, 12, 14, 16],
  target: 10
});

// Inspect individual steps
const step = result.steps[2];
console.log(step.index);        // 2
console.log(step.id);           // "compare_mid"
console.log(step.title);        // "Compare Middle Element"
console.log(step.explanation);  // "mid = 3. array[3] = 8. 8 < 10, search right."
console.log(step.state);        // { array, target, left, right, mid }
console.log(step.visualActions); // [ { type: "movePointer", ... }, ... ]
console.log(step.codeHighlight); // { language: "pseudocode", lines: [5, 6] }
console.log(step.isTerminal);   // false
console.log(step.phase);        // "search"

show(algorithmId, options?)

returns void

Launches an interactive browser-based visualization. Starts a lightweight local HTTP server using only Node.js built-in modules and opens your default browser. Press Ctrl+C to stop.

ParameterTypeDescription
algorithmIdstringAlgorithm identifier (e.g., "self-attention").
optionsShowOptionsOptional configuration object.
options.portnumberTCP port for the local server. Default: 0 (auto-select).
options.openBrowserbooleanWhether to open the default browser. Default: true.
options.inputsRecord<string, unknown>Custom input parameters. Uses defaults if omitted.
Examples 
import { show } from "eigenvue";

// Basic — auto port, opens browser
show("binary-search");

// Fixed port
show("dijkstra", { port: 3000 });

// Custom inputs + no auto-open
show("self-attention", {
  port: 8080,
  openBrowser: false,
  inputs: { tokens: ["The", "cat", "sat"], embeddingDim: 4 }
});
Server Endpoints
RouteResponse
GET /HTML visualizer page
GET /api/stepsJSON — { algorithmId, meta, steps }
GET /api/healthJSON — { status: "ok", algorithmId }
GET /static/*Static assets (JS, CSS, images)

version

string constant

The package version string.

Example 
import { version } from "eigenvue";

console.log(version); // "1.0.0"

Algorithm Catalog

17 algorithms across 3 domains, each with step-by-step visualization, multiple code implementations, educational content, and default example inputs.

Classical Algorithms 7 algorithms

Binary Search

Efficient search in sorted arrays by repeatedly halving the search space.

beginner O(log n) O(1) space 
steps("binary-search", { array: [1,3,5,7,9], target: 5 })

Bubble Sort

Simple comparison-based sorting by repeatedly swapping adjacent elements.

beginner O(n²) O(1) space 
steps("bubble-sort")

Quicksort

Divide-and-conquer sorting using pivot-based partitioning.

intermediate O(n log n) avg O(log n) space 
steps("quicksort")

Merge Sort

Stable divide-and-conquer sorting with guaranteed O(n log n) performance.

intermediate O(n log n) O(n) space 
steps("merge-sort")

Breadth-First Search

Level-order graph traversal using a queue data structure.

intermediate O(V + E) O(V) space 
steps("bfs")

Depth-First Search

Stack-based (or recursive) graph traversal exploring as deep as possible first.

intermediate O(V + E) O(V) space 
steps("dfs")

Dijkstra's Algorithm

Shortest path algorithm for weighted graphs using a priority queue.

intermediate O((V+E) log V) O(V) space 
steps("dijkstra")

beginner O(n * epochs)

Multi-layer neural network with forward propagation through hidden layers.

intermediate O(n * m * k)

interface AlgorithmInfo {

StepSequence

TypeScript 
interface StepSequence {
  readonly formatVersion: number;           // Always 1
  readonly algorithmId: string;             // Algorithm identifier
  readonly inputs: Record<string, unknown>; // Input parameters used
  readonly steps: readonly Step[];          // Ordered steps (min 1)
  readonly generatedAt: string;             // ISO 8601 timestamp
  readonly generatedBy:                     // Generator origin
    | "typescript" | "python" | "precomputed";
}

Step

TypeScript 
interface Step {
  readonly index: number;                       // 0-based position
  readonly id: string;                          // URL-safe identifier
  readonly title: string;                       // Human-readable heading
  readonly explanation: string;                 // Plain-language narration
  readonly state: Record<string, unknown>;     // Complete state snapshot
  readonly visualActions: readonly VisualAction[]; // Rendering instructions
  readonly codeHighlight: CodeHighlight;        // Source code line mappings
  readonly isTerminal: boolean;                // true only on final step
  readonly phase?: string;                     // Optional grouping label
}

VisualAction

TypeScript 
interface VisualAction {
  readonly type: string;          // Action type (e.g., "highlightElement")
  [key: string]: unknown;       // Action-specific parameters
}

CodeHighlight

TypeScript

}

Advanced Usage

Build Custom Visualizations

Use the step data programmatically to build your own visualization layer — for a React app, a CLI tool, a Jupyter widget, or anything else.

TypeScript 
import { steps } from "eigenvue";
import type { Step, VisualAction } from "eigenvue";

const seq = steps("binary-search", {
  array: [2, 4, 6, 8, 10, 12],
  target: 8
});

// Replay all steps for a custom renderer
for (const step of seq.steps) {
  renderStepHeader(step.index, step.title);
  renderExplanation(step.explanation);

  // Process each visual action
  for (const action of step.visualActions) {
    switch (action.type) {
      case "highlightElement":
        highlightArrayCell(action.index, action.color);
        break;
      case "movePointer":
        animatePointer(action.id, action.to);
        break;
      case "showMessage":
        displayMessage(action.text, action.messageType);
        break;
      // Unknown types are silently ignored
    }
  }

  // Highlight source code lines
  highlightCode(step.codeHighlight.language, step.codeHighlight.lines);

  if (step.isTerminal) {
    showFinalResult(step.state);
  }
}

Group Steps by Phase

Steps may include an optional phase field for logical grouping. Use it to organize complex algorithms into sections.

TypeScript 
import { steps } from "eigenvue";

const seq = steps("transformer-block");

// Group steps by phase
const phases = new Map<string, Step[]>();

for (const step of seq.steps) {
  const phase = step.phase ?? "default";
  if (!phases.has(phase)) phases.set(phase, []);
  phases.get(phase)!.push(step);
}

for (const [phase, phaseSteps] of phases) {
  console.log(`\n── ${phase} (${phaseSteps.length} steps) ──`);
  for (const s of phaseSteps) {
    console.log(`  ${s.title}`);
  }
}

Export Step Data to JSON

All step data is fully JSON-serializable — no functions, no circular references, no undefined values.

TypeScript 
import { steps } from "eigenvue";
import { writeFileSync } from "node:fs";

const result = steps("self-attention");

// Save to file
writeFileSync("self-attention-steps.json", JSON.stringify(result, null, 2));

// Send over HTTP, store in database, pipe to another tool, etc.

Batch-Process All Algorithms

Combine list() and steps() to process every algorithm programmatically.

TypeScript 
import { list, steps } from "eigenvue";

for (const algo of list()) {
  const seq = steps(algo.id);
  console.log(
    `${algo.name.padEnd(25)}  ${seq.steps.length} steps  ${algo.timeComplexity}`
  );
}

Error Handling

All errors throw standard Error objects with descriptive messages.

TypeScript 
import { list, steps } from "eigenvue";

// Invalid category
try {
  list({ category: "invalid" as any });
} catch (e) {
  // Error: Invalid category "invalid". Valid categories: classical, deep-learning, ...
}

// Unknown algorithm
try {
  steps("not-real");
} catch (e) {
  // Error: Unknown algorithm "not-real". Use list() to see available algorithms.
}

Key Design Decisions

0

Zero Dependencies

No runtime dependencies. Uses only Node.js built-in modules (http, fs, path, child_process).

D

Dual ESM + CJS

Works with both import and require(). Type declarations for both formats.

R

Immutable Returns

All public API returns use readonly types. list() returns a fresh copy each call.

V

Open Action Vocabulary

New visual action types can be added without breaking existing renderers. Unknown types are silently ignored.

L

Local-First Visualization

Browser visualization requires no internet. The server runs entirely on localhost.

F

Format Versioning

StepSequence.formatVersion enables forward/backward compatibility.
